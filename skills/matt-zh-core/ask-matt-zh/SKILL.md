# 询问马特

你不记得每一项技能，所以要问。

**流程**是通过技能的路径。大多数路径沿着一条**主流**运行，并有两条**入口匝道**并入其中。其他一切都是独立的。

## 主流程：想法→船舶

大多数工作出行的路线。您有一个想法并希望将其实现。

1. **`/grill-with-docs-zh`** — 通过采访加深想法。当您**拥有代码库**时，请从这里开始：它是有状态的，保留在“CONTEXT.md”和 ADR 中学到的内容。 （没有代码库？使用 `/grill-me-zh` — 请参阅独立版。）
2. **分支 — 你能解决对话中的每个问题吗？** 如果问题需要一个可运行的答案（状态、业务逻辑、你必须看到的 UI），请绕道原型，在两个方向上通过 **`/handoff-zh`** 桥接（请参阅交叉会话）：
   - **`/handoff-zh`** 退出，然后针对该文件打开一个新会话，
   - **`/prototype-zh`** 用一次性代码回答问题，
   - **`/handoff-zh`** 返回您所学到的内容，并从原始想法线程中引用它。
3. **分支 — 这是多会话构建吗？**
   - **是** → **`/to-prd-zh`**（将线程变成 PRD）→ **`/to-issues-zh`** （将 PRD 拆分为可独立抓取的问题）。因为这些问题是独立的，**每个问题之间都有清晰的上下文**：每个问题开始一个新的会话，并通过向其传递 PRD 和要处理的单个问题来启动 **`/implement-zh`**。
   - **否** → **`/implement-zh`** 就在这里，在同一个上下文窗口中。

### 环境卫生

将步骤 1-3 保留在**一个不间断的上下文窗口**中 - 在“/to-issues-zh”之前不要压缩或清除 - 因此烧烤、PRD 和问题都建立在相同的思维之上。然后，每个“/implement-zh”都会重新开始，从问题开始工作。

对此的限制是 **[智能区域](https://www.aihero.dev/ai-coding-dictionary/smart-zone)**：该窗口（最先进模型上的约 120k 代币）在该窗口内模型仍然可以进行敏锐的推理。如果会话在“/to-issues-zh”之前接近它，请不要继续降级 - “/handoff-zh”并在新线程中继续。

## 入口匝道

生成工作的起始情况，然后合并到主流中。

- **错误和请求堆积** → **`/triage-zh`**。它通过分类角色转移问题并产生代理就绪的问题，这些问题稍后会被处理。

  分类仅适用于**不是您创建的**问题 - 错误报告、传入的功能请求、任何原始的内容。 `/to-issues-zh` 产生的问题已经可供代理使用，因此**不要对它们进行分类**。

## 代码库健康状况

不是功能工作——维护。

- **`/improve-codebase-architecture-zh`** — 只要有空闲时间就运行，以保持代码库适合代理操作。它会带来深化的机会；选择一个_产生一个想法_您可以将其纳入“/grill-with-docs-zh”的主要流程。

## 交叉会话

- **`/handoff-zh`** — 当线程已满或您需要分支（例如进入 `/prototype-zh` 会话）时，这会将对话压缩到一个 markdown 文件中。您不需要继续原地不动 - 您**打开一个新会话并引用该文件**来传递上下文。它是任一方向上下文窗口之间的桥梁。当您想要**新会话**但需要**保留当前对话**时，请使用它。
- **`/compact`**（内置）- 保持**相同的对话**，让之前的回合进行总结。当您不介意丢失逐字历史记录时，可以在**阶段之间的有意中断**时使用它。不要在中期压缩——代理可能会迷失方向。 `/handoff-zh` 叉子； `/compact` 继续。

## 独立

完全脱离主流。- **`/grill-me-zh`** — 与“/grill-with-docs-zh”相同的无情面试，但当你**没有代码库**时。无状态：它在本地不保存任何内容，不构建“CONTEXT.md”。利用它来完善不在存储库中的任何计划或设计。
- **`/teach-zh`** — 使用当前目录作为有状态工作区，通过多个会话学习概念。
- **`/writing-great-skills-zh`** — 良好写作和编辑技能的参考。

## 前提条件

**`/setup-matt-pocock-skills-zh`** — 在第一个工程流程之前运行，以配置其他技能假定的问题跟踪器、分类标签和文档布局。自定义问题跟踪器也可以工作。
