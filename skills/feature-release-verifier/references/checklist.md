# Feature Release Verifier Checklist

- [ ] Verification brief includes feature, scope, risk, runtime target, packaged requirement
- [ ] Verification brief includes impact tags and acceptance keywords when provided
- [ ] Selected lanes are explicitly justified
- [ ] Baseline commands are recorded (`typecheck`, `lint`, and `build` when required)
- [ ] Playwright-first lane was used for user-visible behavior unless explicitly not applicable
- [ ] TESTING_KB rules are enforced where applicable:
  - [ ] network-first / event-first synchronization
  - [ ] fixture ownership / setup centralization
  - [ ] canonical Playwright output stays under `tests/test-results/`
  - [ ] sensitive artifact handling is called out for real-runtime lanes
- [ ] Failed Playwright lanes include artifact paths:
  - [ ] screenshot
  - [ ] trace
  - [ ] error-context
  - [ ] video (when generated)
- [ ] Worker outputs include a consistent schema:
  - [ ] commands
  - [ ] required flag
  - [ ] skip reason
  - [ ] primary failure signal
  - [ ] failing test title
  - [ ] artifact paths
  - [ ] rerun command
  - [ ] bug location hint
- [ ] Worker outputs record `PASS` / `FAIL` / `SKIP` with reasons
- [ ] `SKIP` never masquerades as `PASS`
- [ ] Required lanes that are skipped are surfaced as blockers or an explicit non-pass release gate
- [ ] Artifact manifest includes existence checks and sensitivity/redaction notes where needed
- [ ] Report separates:
  - [ ] source-level verification
  - [ ] real-runtime verification
  - [ ] packaged verification
  - [ ] remote packaged verification
- [ ] Final release decision is one of:
  - [ ] `PASS`
  - [ ] `PASS-WITH-CONCERNS`
  - [ ] `FAIL`
