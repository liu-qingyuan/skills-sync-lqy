# Mermaid 可视化

## 工作流

1. 理解内容和使用场景，识别实体、关系、顺序、层级、状态和对比。
2. 选择最能表达内容结构的图表类型；无法判断时使用流程图。
3. 应用用户指定的布局、细节和样式；未指定时使用默认值。
4. 生成 Mermaid 代码，突出主路径，合并琐碎节点，避免装饰性复杂度。
5. 验证语法和可读性后，使用 `mermaid` 代码围栏输出。

## 图表选择

| 内容结构 | 图表类型 |
| --- | --- |
| 步骤、决策、工作流、系统架构 | 流程图 `flowchart TB/LR` |
| 迭代、反馈、持续改进 | 带反馈连接的流程图 |
| 两个或多个方案的对比 | 使用并行分组的流程图 |
| 父子层级、知识分类 | 思维导图 `mindmap` |
| 参与者随时间发生的交互 | 时序图 `sequenceDiagram` |
| 状态和生命周期转换 | 状态图 `stateDiagram-v2` |

## 默认配置

- 布局：纵向 `TB`；横向展示更清楚时使用 `LR`。
- 细节：`standard`；只保留理解关系所需的标签和说明。
- 样式：`professional`；使用一致的语义色彩和清晰层级。
- 兼容性：优先使用 GitHub 和 Obsidian 均支持的 Mermaid 语法。

用户可指定：

- 方向：`TB`、`LR`、`RL`、`BT`。
- 细节：`simple`、`standard`、`detailed`、`presentation`。
- 样式：`minimal`、`professional`、`colorful`、`academic`。
- 标题、步骤编号和图例。

## 语法护栏

- 每个 Mermaid 代码块都以图表类型声明开头，例如 `flowchart TB`。
- 节点和子图使用稳定 ID；显示名称只用于标签。
- 名称含空格的子图使用 `subgraph id["显示名称"]`。
- 连接始终引用 ID，不直接引用显示文本。
- 节点文本避免“数字 + 句点 + 空格”，防止被识别为 Markdown 列表。
- 对含空格或特殊字符的文本使用引号；避免容易引发解析歧义的字符。
- 节点文本不使用 Emoji，改用文字标签或颜色编码。
- 输出前验证所有 Mermaid 代码块，不能把未验证的语法当作示例。

需要节点形状、箭头、子图、样式、布局、常用模式或排错细节时，读取 [Mermaid 语法规则参考](references/syntax-rules.md)。

## 输出

- 直接给出可渲染的 Mermaid 代码。
- 仅在有助于理解时补充简短的结构说明。
- 用户指定平台时，按该平台支持的 Mermaid 版本选择语法。
