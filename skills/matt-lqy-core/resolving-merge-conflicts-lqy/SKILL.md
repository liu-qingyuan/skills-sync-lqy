---
name: resolving-merge-conflicts-lqy
description: 用于解决正在进行中的 git merge 或 rebase 冲突。
---

1. **查看合并/变基的当前状态**。检查 git 历史记录和冲突文件。

2. **找到每个冲突的主要来源**。深刻理解每次改变的原因以及初衷是什么。阅读提交消息，检查 PR，检查原始问题/票据。

3. **解决每个问题。** 只解决可以从代码、提交消息、PR、issue 或现有契约中明确推导出的冲突。保留的是双方仍然有效的业务意图和用户可见需求，不是机械保留旧字段、旧 API 形状、兼容分支或过时实现。

   如果冲突涉及字段/API 去留、兼容层是否保留、迁移方向、用户可见行为变化，或双方意图无法同时满足，**不要自行取舍**。暂停修改，整理证据和可选方案，交给人类确认后再继续。

   人类确认前，不要为了折中重新引入已废弃字段，不要发明新行为，不要完成合并提交；也不要用 `--abort` 逃避冲突，除非人类明确要求。

4. 发现项目的**自动检查**并运行它们 - 通常是类型检查，然后测试，然后格式化。修复合并损坏的所有内容。

5. **完成合并/变基。** 暂存所有内容并提交。如果进行变基，请继续变基过程，直到所有提交都变基。
