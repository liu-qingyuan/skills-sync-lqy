# 领域文档

此仓库使用单上下文领域文档布局。

## 探索前读取

- 根目录 `CONTEXT.md`：统一术语表。
- `docs/adr/`：与当前工作相关的架构决策记录；目录不存在时安静地继续。

不要预先创建空的 ADR 目录或文档。术语或难以逆转的架构决策真正明确时，由 `domain-modeling-lqy` 按需创建或更新。

## 文件结构

```text
/
├── CONTEXT.md
├── docs/
│   └── adr/        # 按需创建
└── skills/
```

## 术语规则

issue 标题、spec、Ticket、重构提案和测试名称应使用 `CONTEXT.md` 中定义的术语，并避免其中明确列出的同义词。

如果需要的概念尚未出现，先判断它是项目中的真实领域概念，还是临时实现语言；只有前者才应通过 `domain-modeling-lqy` 加入术语表。

## ADR 冲突

如果提案与现有 ADR 冲突，明确指出冲突及重新讨论的理由，不要静默覆盖既有决定。
