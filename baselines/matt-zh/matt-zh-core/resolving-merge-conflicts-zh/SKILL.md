---
name: resolving-merge-conflicts-zh
description: 用于解决正在进行中的 git merge 或 rebase 冲突。
---

1. **查看合并/变基的当前状态**。检查 git 历史记录和冲突文件。

2. **找到每个冲突的主要来源**。深刻理解每次改变的原因以及初衷是什么。阅读提交消息，检查 PR，检查原始问题/票据。

3. **解决每个问题。** 保留双方仍然有效的业务意图和用户可见需求，而不是机械保留双方的旧字段、旧 API 形状、兼容分支或过时实现。如果一方变更是在迁移、重命名或删除旧结构，优先尊重新结构；只有存在明确兼容契约或仍有调用方依赖时，才保留兼容层，并标注后续退役路径。如果意图不兼容，请选择与本次合并/变基既定目标相匹配的方案并记下权衡。**不要**为了折中而重新引入已废弃字段，也**不要**发明新行为。必须做出明确取舍；不要用 `--abort` 逃避冲突。

4. 发现项目的**自动检查**并运行它们 - 通常是类型检查，然后测试，然后格式化。修复合并损坏的所有内容。

5. **完成合并/变基。** 暂存所有内容并提交。如果进行变基，请继续变基过程，直到所有提交都变基。
