# 术语表 — 培养出色的技能

是什么让一项技能变得伟大的领域模型。有一种技巧可以将决定论从随机系统中剔除出来。根本美德是**可预测性**，下面的每一项都是其杠杆。这是 [`writing-great-skills`](SKILL.md) 公开的参考文献。

这些术语按轴分组：**调用**（如何达到技能）、**信息层次结构**（其内容如何排列）、**指导**（如何塑造agent 的运行时行为）和**修剪**（如何保持精简）。每个**故障模式**都位于解决它的杠杆旁边，标记为_故障模式_。

**任何定义中的粗体术语**本身均在本术语表中定义；通过标题找到它们。

## 可预测性

技能使agent 在每次运行中以相同方式表现的程度 - 相同的过程，而不是相同的输出（头脑风暴技能应该_可预测_发散；它的标记有所不同，但其行为却没有）。所有其他术语所服务的根本优点是成本和可维护性是它的症状，而不是竞争对手。

_避免_：一致性、可靠性、鲁棒性、输出决定论

## 调用

如何获得一项技能——以及你为选择付出的两重负担。

### 模型调用

保留其 **描述** 字段的技能，因此agent 可以看到它并自主触发它 - 并且人类仍然可以输入其名称，因此模型调用始终_包括_用户可达。不存在仅模型状态：描述仅_添加_agent 发现，永远不会删除人类的。每回合支付永久的**上下文负载**以换取这种可发现性。可通过其他技能达到，因为使其可被agent 发现的描述使其可调用。内容全部为**引用**的模型调用技能也是共享引用的一个家：另一个技能可以调用它，因此多个技能所需的引用位于一个地方。仅当 agent 必须自己触发该 skill 时才选择模型调用；如果除了手动之外它永远不会触发，则删除描述并且不支付上下文负载。

_避免_：能力、工具、能力

### 用户调用

其 **描述** 被剥离的技能 - 对 agent 不可见，只能通过人类输入其名称才能访问（user-_only_，其中 **model-invoked** 是 user-_and-agent_）。以 agent 可发现性换取零**上下文负载**。因为它没有描述，只有人类能触发它：没有其他 skill 能调用它。

_避免_：程序、工作流、命令

### 描述

该技能的机器可读触发器，以及一个**上下文指针**（一个**模型调用的**技能）被迫始终保持加载状态。它的存在只是调用轴：保留它并且该技能是模型调用的（并且可以由其他技能访问）；删除它，该技能将由**用户调用**，只有人类才能访问。模型调用技能的**上下文负载**的来源。

_避免_：前言、摘要

### 上下文指针

保存在 agent 上下文中的引用，用来指向上下文外的材料，并编码什么时候应该读取它。**描述**是顶层上下文指针（上下文窗口 → skill）；指向公开文件的链接是下一层的同类对象。决定 agent 何时读取、能否稳定读取的，是指针的措辞，而不是目标文件本身。必要材料藏在措辞薄弱的指针后面，就是方差问题：先修指针措辞，仍不稳定时再把材料内联。

_避免_：链接、引用、导入

### 上下文加载

**模型调用**技能对agent 的上下文窗口施加的成本 - 它的**描述**，始终加载，消耗token和注意力。 **用户调用的**技能通过没有描述而逃脱，并且阻止分裂成更多模型调用的技能。

_避免_：token成本、上下文膨胀

### 认知负荷**用户调用的**技能强加给人类的成本——他们必须牢记在心：存在哪些技能以及何时使用每种技能（人类是索引）。 **模型调用**通过 agent 可发现性来消除的成本，并阻止分裂成更多用户调用的技能。这不是一个需要最小化的成本：这是人类agent 的代价，也是某些技能仍由用户调用的原因。把它花在人类判断力重要的地方；将其删除。

_避免_：人力指数、负担、开销

### 路由器技能

一种**用户调用型** skill，职责是指向其他用户调用型 skill：说明每个 skill 的名字以及什么时候使用它。这样人只需要记住一个入口，而不是记住一堆 skill。它只能提示用户使用哪些 skill，不能替用户调用它们：用户调用型 skill 没有**描述**，因此只有人类能触发。它用于降低用户调用型 skill 增多时带来的**认知负荷**。

_避免_：调度程序、菜单、注册表、索引、路由器程序

### 粒度

你对技能的划分有多细。更精细的划分花费了两种负载之一：更多的**模型调用**技能花费**上下文负载**（更多的描述挤满窗口并争夺注意力）；更多的**用户调用的**技能花费**认知负荷**（更多的人类需要记住和达到）。两个削减指导划分。通过**调用**，分离出模型调用的技能，其中您有一个独特的**引导词**来触发它 - 您在提示中实际使用的触发词。通过**序列**，分割一系列**步骤**，其中步骤的**完成后步骤**需要隐藏，因为将其隔离在其自己的上下文中可以清除后续内容。请注意相反的情况：合并序列会将每个步骤的完成后步骤暴露给后续步骤，从而导致过早完成。

_避免_：分块、模块化

## 信息层次结构

技能的内容是如何安排的，以及每个部分在阶梯上的位置。

### 信息层次结构

skill 内容根据 agent 对材料的即时需要程度排序 - 一个梯子，由两次剪切产生：文件内或指针后面，以及步骤或参考。梯级：

- **步骤** — 文件内，主要
- **参考**，文件内 — 次要
- **参考**，公开 - 在**上下文指针**后面

没有**步骤**的技能仅使用底部的两个梯级——通常是合法的平坦同行集（例如，一个梯级上的每条审查规则），这是一个很好的安排，而不是一种气味。层次结构独立于调用：技能可以由模型或用户调用，无论它是所有步骤、所有参考还是两者兼而有之。当一项技能有步骤时，应该公开的文件内参考将其隐藏起来，并将对它们的关注变成了抛硬币——一个方差杠杆，而不仅仅是一个易读性杠杆。保持梯子顶部清晰可见；尽你所能地把它压下去。

_避免_：结构、组织、布局

### 步骤

agent 执行的一组有序操作。当 skill 有步骤时，步骤就是内容的主层，也是最有资格留在 `SKILL.md` 里的部分。并非每个 skill 都有步骤：skill 可以全是步骤（如 `tdd`），也可以全是**参考**（如 review），或两者兼有；这与调用方式无关。每个步骤都会以明确或模糊的**完成标准**结束。

_避免_：工作流、说明、编排

### 参考

agent 按需引用的材料——定义、事实、参数、示例、条件指令。当一项技能有**步骤**时，它对它们来说是次要的；当一项技能没有时，它就是全部内容；或者它完全不属于任何技能——请参阅**外部参考**。通过**上下文指针**达到，并且是**渐进式披露**的主要候选项。

_避免_：支持材料、文档、背景

### 外部参考**参考**存在于技能系统之外——一个普通文件，没有**描述**，没有**步骤**，不可调用——任何技能都可以指向。用于共享参考的主页不需要自行触发，并且是两个**用户调用的**技能可以使用的唯一共享主页，因为两者都没有描述，因此都不能触发对方。

_避免_：文档、资源、知识库

### 渐进式披露

将 **参考** 沿着梯子移动 - 从 SKILL.md 移出并位于 **上下文指针** 后面 - 以便顶部保持清晰。主要不是token优化；这就是**信息层次结构**的保护方式。由**分支**许可：披露只有某些分支需要的内容，内联每条路径需要的内容，如果指针在必备材料上不可靠地触发，则锐化其措辞，只有在失败时才将其拉回内联。

_避免_：延迟加载、分块

### 主机托管

把 agent 需要一起理解的材料放在同一个地方：概念定义、规则和注意事项应在同一标题下，而不是散落在文件各处。它是**信息层次结构**在单个文件内部的配套原则：层次结构决定一块内容放在哪一层；一旦 agent 到达这一层，协同定位决定旁边应该放什么。**参考**正文没有唯一正确格式；判断标准是：skill 应该读起来像写给 agent 的文档，成组材料要能一次读懂，散落材料则不行。它不同于**重复**：重复是在两处表达同一含义；协同定位问题是同一含义被拆散到多个地方。

_避免_：分组、聚类、内聚

### 蔓延

_失败模式。_技能太长——SKILL.md 中的行太多——与它们是否过时或重复无关。即使是全活、全独特的技能也可能会蔓延。它需要可读性（agent 在采取行动之前需要阅读更多内容，并且注意力会随着多余的内容而变薄）、可维护性（每多一行都是为了保持**相关**而多加一行）和token。解决方法是**信息层次结构**：将**引用**推到**上下文指针**后面，并按**分支**或序列进行分割，以便每个路径仅携带其需要的内容。与**沉积物**（陈旧堆积的长度）和**重复**（重复含义的长度）不同，蔓延就是长度本身，无论其原因如何。

_避免_：臃肿、长度、大小、冗长

## 转向

塑造 agent 运行时行为以实现**可预测性**的杠杆。

### 分支

调用技能的独特方式（技能处理的情况）因此不同的运行会采取不同的路径。一项技能的步骤多，则分支也多；线性的则没有。

_避免_：路径、案例、分叉

### 引导词

一个紧凑概念，也叫 _Leitwort_，通常已经存在于模型预训练中，agent 运行 skill 时会围绕它组织思考。它用尽可能少的 token 调动模型已有先验，从而编码行为原则（例如 _lesson_、_zone of proximal development_、_fog of war_、_tracer bullet_）。它不是反复写一整句规则，而是反复使用一个关键词；这个词会在 skill 中积累分布式定义，并锚定一整片行为区域。你当然可以创造新词，但虚构词无法借用预训练先验——你必须用定义来支付模型本来免费提供的成本。因此先找已有词。引导词能从两方面服务**可预测性**：在正文里，它锚定**执行**，让 agent 每次看到同一概念时采取同类行为；在扁平参考里，它聚焦 agent 应寻找的事项，让每次运行都触发正确检查。在**描述**里，它锚定**调用**；而且不只在 skill 内部生效：当同一个词出现在提示、文档和代码库中，agent 会把这套共享语言连接到 skill，从而更稳定地触发它。描述里要使用你真正会在需要该 skill 时使用的引导词。

_避免_：关键词、术语、主题

### 完成标准

告诉 agent 一个工作单元何时完成的条件——它判断的目标。有两个特性使它成为一种杠杆，而不仅仅是一种质量。它的**清晰度**（agent 能区分完成和未完成吗？）抵制**过早完成**——模糊的界限（“达成理解”）让 agent 宣布完成并滑到下一步；该轴需要“步骤”来咬合，因为过早完成是步骤之间的失败。它的**需求**（需要多少）设置**腿部工作**——“每个修改的模型都占到”强制彻底的工作，而“生成更改列表”则不然——而且这个轴是_not_步骤绑定的：它也可以绑定平面参考主体，这就是没有步骤的技能仍然带有穷举栏的方式（“应用的每条规则”）。最严格的标准既是可检查的又是详尽的。

_Avoid_：完成条件、退出条件、停止规则

### 跑腿

agent 在幕后一步完成的工作 - 读取文件、探索代码库、进行更改、挖掘所需内容，而不是卸载给用户。它存在于步骤结构之下：从未被写成自己的步骤，隐藏在措辞中，由 agent 而不是 skill 控制。步骤内对应于**完成后步骤**'跨步骤拉动。由**主导词**（_全面_、_彻底_）或**完成标准**提出，要求工作详尽无遗——包括应用于平面参考的需求轴，这就是驱动平面参考技能覆盖其所有梯级的原因。当需求缺失或**过早完成**缩短步骤时，就会变得稀薄。

_避免_：范围、努力、勤奋、覆盖范围

### 完成后步骤

当前步骤之后的**步骤**。可见，它们将智能体向前拉入**过早完成**——它看到的越多，拉力就越强；防御方法是将步骤序列分成两部分来隐藏它们。

_避免_：地平线、战争迷雾、前瞻

### 提前完成_失败模式。_在真正完成之前结束当前步骤，因为agent 的注意力转移到正在完成而不是工作上。步骤之间的失败：它需要**步骤**才能发生——没有步骤的提前退出的技能并不是过早完成，而是在未满足的需求下的薄弱**跑腿工作**。两种力量之间的拉锯战：可见的**完成后步骤**（向前拉动）和**完成标准**的清晰度（阻力——一个尖锐的、可检查的条保持；一个模糊的条让路）。模糊性是必要条件：无论有多少后续步骤可见，尖锐的边界都可以抵抗拉力，因此从不急速的步骤不需要防御。两个杠杆支撑着一个台阶，但要按顺序接触它们：**首先锐化边界**——它是本地的且便宜。只有当标准不可约地模糊_并且_您实际上观察到匆忙时，您才**隐藏后续步骤** - 并且隐藏仅适用于真实的上下文边界（用户调用的交接或sub-agent调度；内联模型调用的调用将后续步骤保留在上下文中并且不清除任何内容）。腿部工作量薄弱的原因之一，但又与之不同：即使一个步骤完全完成，腿部工作量也可能很薄弱。

_避免_：过早关闭、匆忙、仓促、走捷径

### Negation

_失败模式。_ 用禁止来 steering：告诉 agent 不要做什么。它会把被禁止的行为拖进上下文，让它_更_可用，而不是更少。_Don't think of an elephant_，结果 elephant 占满画面；_never write verbose comments_，verbosity 这个 pattern 刚被 agent 读到。否定词是弱 modifier，被强激活的概念压过，所以禁令会半读成“做这件事”。它的**引导词**就是那个 _elephant_：任何 prohibition 命名进框架的东西。修法：prompt the **positive**，描述目标行为（例如 “write one-line comments”），让被禁止的东西不被说出口。只有无法正向表达的硬 guardrail 才值得保留 prohibition；即便如此，也要配上正向目标，让注意力落在要做什么上。

_避免_：ironic rebound、don't-prompting、the pink elephant

## 修剪

保持技能精益——每一种补救措施都与它所解决的失败相匹配。

### 单一事实来源

每个含义都恰好位于一个权威位置的理想状态，因此技能行为的更改就是一个位置的更改。 **重复**是其违规行为。

_避免_：家，规范位置

### 重复

_失败模式。_相同的含义给出了多个**单一事实来源**。它需要维护（改变一个地方，你必须改变其他地方），成本token，并夸大重要性——重复一个含义会使其在阶梯上的权重超过其真实排名。 **主导词**的意外反转，通过重复一个标记而不是其含义来故意引起注意。

_避免_：重复、冗余

### 相关性

台词是否仍然与技能的作用有关——保留什么的镜头。一条线失去了相关性，要么是因为它从不承担任务（仅仅是阐述，或者是应该披露的**分支**），要么是因为它变得陈旧：随着它所描述的行为或世界的变化而逐渐过时。较短的技能更容易保持相关性，因为检查每行的成本更便宜。与**无操作**不同：相关性询问一条线是否与任务有关，而不是它是否改变行为。

_避免_：承重、陈旧、新鲜

### 沉积物

_失败模式。_旧内容层层堆积在技能中，永远不会被清除，因为添加感觉安全，删除感觉有风险——因此陈旧和不相关的行会累积，你必须深入了解它们才能找到仍然存在的内容。任何没有修剪规则的技能的默认命运； **相关性**的缓慢侵蚀，与**重复**的重复含义相反。

_避免_：堆积、膨胀、腐烂、腐烂

### 无操作

_失败模式。_ 一条不会改变行为的指令，因为模型默认已经会这么做；你只是在花 token 告诉 agent 它本来就会做的事。测试方式：这行话会让行为区别于默认行为吗？一行内容可以完全**相关**，但仍然是 No-Op。让**引导词**变得便宜的同一套先验，也会让 No-Op 变得毫无价值。引导词是_技术_；No-Op 是对某一行的_判断_，两者会交叉。弱到无法改变默认行为的引导词就是 No-Op（如果 agent 默认已经足够彻底，“要彻底”就是 No-Op）；修法是换成判断更强的词（如 _relentless_），而不是换一种技术。因此 No-Op 测试——它是否改变默认行为？——也用于判断一个引导词值不值得重复。这取决于模型，不取决于读者感受：如果两个人争论某行是否 No-Op，应通过运行 skill 来判断，而不是靠辩论。

_避免_：冗余指令、重述显而易见的内容、繁琐的说明
